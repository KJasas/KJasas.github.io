Football

Our goal is to collect some specific information from sports statistics site. The written program visits unique pages of the site and collects specific part of it's html code.
Let's say that we want to collect information about the attendance of the 2015/2016 Premier Leagues (football/soccer) matches. Every match has an url, example

https://www.[SportsSite].com/Match/829783

The number "829783" is matches unique ID. We look at the first and the last season matches' ID's, which are 829517 and 829799 accordingly. We know that these matches go sequentially, so almost (some information about matches may be changed long after, so the match can get non-sequential ID) all matches should be between these numbers. To be careful, we widen the range, let's say 829500 to 829850. The written R code tries every url in this range. If an url exists, the code checks in its html code if the season and league is right. If so, it searches the html and saves the playing teams names, date of match and number of attendance.
  • Some matches have ID's which are way beyond the range, so some missing data must be collected by hand.
  • The site [SportsSite].com has safeguard against automatic data scraping - from time to time it requires manually going to the site to prove that "you're a human". Sometimes brute force works (trying 10 times every url, checking if the url is read as empty, which happens when safeguard appears), sometimes not (every other url in the range is read as empty). So human interference is needed - stopping the code, checking in as a human, resuming the code.
  • Reading all seasons of all leagues at once.
  • Collecting more information from each url.
  • Somehow finding a better way around the safeguard.
